**日期：** 2026-07-18  
**原则：** 页面有 ≠ 链路通 ≠ 真出片可用 ≠ 旗舰质感。分数只认代码 + 已有 live 门禁证据。  
**定位：** Betty = **多模型导演工作室**（Yapper 类），不是 Runway/Kling/HeyGen/Midjourney 一等公民模型厂。  
**关联证据：** `OPTIMIZE_GATE_VERIFICATION.md`（5/5 CAP）· `FULL_FEATURE_VERIFICATION_EVAL.md` · `GAP_CLOSURE_DEEP_ANALYSIS.md` · `WORLD_CLASS_BENCHMARK.md`

## 0. 一句话裁决

| 问题 | 答案 |
|------|------|
| 是否「每个细节都继续抠」？ | **否。** 继续优化应集中在 **ROI 最高的可自建层（L2/L3）**；L1 上游天花板换模/商务才质变。 |
| 当前相对顶级整体 | **编排交付 ~75–80%** · **视听旗舰质感 ~55–65%** · **投放工厂 ~45–55%** |
| 最该继续做 | 变体画廊、商用 BGM、角色一致性产品化、失败重试/SLA UX |
| 不该再硬抠 | InfiniTalk 默认、假宣称 Act-One/HeyGen 口型、lab 模型虚标货架 |

## 1. 评估坐标系（像素级四层）

```
L0 像素/帧观感     ← 材质、脸、口型、运动伪影（主要由上游模型决定）
L1 单能力链路       ← 文生图/视频/口播/Motion/换脸… 是否稳定真出片
L2 导演编排         ← 场景路由、画幅、多镜、身份、合成包装
L3 投放/工厂完成度  ← 曲库、字幕花字、批量变体、A/B、分析回流
L4 商业与信任       ← 计费、SSO、目录诚实、SLA、社区飞轮
```

**禁止混层对标：** 用 L2「有成片按钮」去宣称追平 HeyGen L0 口型 = 虚假 parity。

对标锚点：

| 锚点平台 | 代表维度 |
|----------|----------|
| **Yapper** | 工具矩阵 / Agent 导演 |
| **HeyGen / Hedra** | 数字人口型 |
| **Arcads / Creatify** | UGC / 广告投放工厂 |
| **Runway / Kling / Luma** | 视频运动 / Motion |
| **CapCut** | 字幕花字 / 剪辑完成度 |
| **HeadshotPro / Photoroom** | 写真 / 产品静图 |
| **ShortMax** | 微短剧表演与剧本 |
| **Advantage+** | 批量创意变体 |

## 4. 最好方案与优化建议（按「是否继续」决策）

### 4.1 必须继续（高 ROI · 可自建）

#### A. 投放包装升级（关 G02/G03）— **最值得**

**方案：**
1. 环境变量接入 4–8 条**可商用** BGM URL（按 soft/upbeat/cinematic/drama），fixtures 作 fallback  
2. compose 增加简单侧链 ducking（人声优先）  
3. 字幕模板扩展到 8–12：字号/描边/位置/入场（不必一次 CapCut 全量）  
4. 口播路径保持 `keep_source_audio`，广告路径强制 BGM+CTA  

**预期：** 观感从「demo 床」→「可试投」再抬 **15–25%**；不碰 L0 模型。  
**工作量：** 中（合成器 + 素材配置 + 回归门禁）。

#### B. 变体工厂后半段（关 G04）— **最值得**

**方案：**
1. FE：变体卡片并行跑 2–3 条 → **并排成片画廊**（缩略图+轴标签）  
2. 后端：`/director/variants/run` 批量 job + 进度聚合  
3. 可选：人工点选「采用」写回 session  

**预期：** 对广告团队感知从「有 API」→「有工厂」；补齐 Gap5 剩余一半。  
**工作量：** 中高（FE 为主）。

#### C. 身份一致性产品化（关 G05）— **值得**

**方案（当前栈最优，不幻想 LoRA 训练平台）：**
1. 计划层暴露「身份锁强度」：off / edit-variant（现有） / 强制同 hero  
2. 多镜出片后自动拼 **身份对比条**（hero | shot2 | shot3 帧）供人眼验收  
3. 中期：接支持 reference-pack 的上游（若 KIE 上架）  

**预期：** 多镜「同一人/同一产品」投诉下降；对标仍非角色库。  
**工作量：** 中。

#### D. 可靠性 UX（关 G11）— **值得**

**方案：** 步骤级重试、队列 ETA、失败自动换 SKU（已有部分）+ 前端明确「正在排队/可取消」。  
**预期：** 转化与信任，不直接提画质分。

---

### 4.2 有条件继续（场景价值高，但别像素死磕）

| 项 | 建议 | 停止线 |
|----|------|--------|
| UGC 原生感 | 提示词库 + 可选「真人参考图」强制 i2v；不要再抠 sine BGM | 不做真人演员签约前，不宣称 Arcads |
| 微短剧 | 加「钩子→冲突→反转」三拍剧本模板 + 对白 TTS | 不做表演模型前，分数很难 >4.0 |
| 动漫角色锁 | 同 G05；加风格 seed 硬化 | 不做专模 |
| 文生视频默认 | 按场景锁最佳 SKU（已有）+ 负面提示库 | 不刷 lab 模型充数 |
| Extractor | 补主流站解析 | 不做全网爬虫 |
| 换脸 | 若商务需要，换真 face-swap 后端；否则文案诚实为「AI 编辑换脸」 | 禁止 InsightFace 级营销 |

---

### 4.3 明确停止 / 降级（继续抠 = 烧预算）

| 项 | 原因 | 正确动作 |
|----|------|----------|
| **默认 InfiniTalk** | 实测超时/内部错 | 保持 Kling；Studio 手动开关 |
| **宣称追平 HeyGen 口型** | L0 天花板 | 只卖「可用口播」+ 响度/构图 |
| **宣称 Act-One / 像素换脸** | 能力不符 | 文案与 badge 保持 Native/诚实 |
| **把 20 个 lab SKU 当货架** | 422/不可售 | 维持 verified=9；逐个 live 验证再上架 |
| **在无授权下「像素级调色」死磕商业片** | 缺 LUT/调色流水线 ROI 低 | 先做 BGM+字幕包装 |
| **社区飞轮当生成优化** | 增长问题 | 单独立项，不进生成像素迭代 |

## 5. 「是否继续优化每一个细节」决策框架

对每个细节用三问：

1. **用户付钱时是否第一眼感知？**（口型、BGM、字幕、竖屏、失败）  
2. **当前栈能否在不换模下实质提升？**（能→做；不能→商务/换模）  
3. **是否已有门禁证明主路径可用？**（可用→优先打磨交付；不可用→先修洞）

### 推荐路线图（只做三刀）

| 顺序 | 刀 | 关差距 | 预期平台提升 |
|------|----|--------|--------------|
| **1** | 商用 BGM + 字幕模板 + ducking | G02/G03 | 投放观感质变 |
| **2** | 变体并排画廊 + 批量跑 | G04 | 广告产能质变 |
| **3** | 身份对比条 + 锁强度开关 | G05 | 多镜可信度 |

做完这三刀后：**暂停像素级发散**，转入：
- 上线商业（Stripe/SSO）  
- 周检 live（图/视/口播）  
- 按真实用户差评再开下一刀  

**不要**同时开：InfiniTalk 深潜、Act-One 对齐、全量 lab 上架、短剧表演大模型自研。

## 6. 刷新后的分层健康度

| 层 | 健康度 | 说明 |
|----|--------|------|
| L0 像素/帧 | **C** | 受 Kling/Seedance/GPT-Image 约束；口型/运动仍是短板 |
| L1 单能力链路 | **B-** | 主路径可出片；Motion/换脸需诚实表述 |
| L2 导演编排 | **B+** | 八场景、画幅、包装、身份、变体门禁已过 |
| L3 投放工厂 | **C+** | 有雏形；曲库/花字/画廊未齐 |
| L4 商业信任 | **C** | 目录诚实好；收款/SSO/飞轮未齐 |

**相对「可上线试投工作室」：** 已够用。  
**相对「全球旗舰生成质量」：** 未够，且不应靠无限抠编排假装够。

---

## 7. 给决策者的结论

1. **平台生成能力已从「能演示」进入「能试投」区间**（导演八功能 + 包装门禁）。  
2. **与顶级的剩余差距，一半在可买/可配的交付层（BGM、字幕、变体 UI），一半在买不到的模型层（口型、运动、角色库）。**  
3. **继续优化：要，但不要「每一个细节」。** 收敛到 §5 三刀；其余细节进入停抠清单。  
4. **正式应用标准：** 任何新优化必须带 `optimize_gate` 式真实验证；未验证不得写入「已追平」。
