## 4. 分层差距（相对全球顶级）

```
┌──────────────────────────────────────────────────────────┐
│ L3 投放工厂：曲库 / 字幕样式 / CTA A-B / 批量变体 / 分析   │  ← 差距最大（可自建）
├──────────────────────────────────────────────────────────┤
│ L2 导演编排：画幅 / 包装 / 身份 / 分轨 / 合成               │  ← 本轮已大幅对齐
├──────────────────────────────────────────────────────────┤
│ L1 上游模型：运动 / 口型 / 身份一致性 / 材质物理           │  ← 天花板（换模/混合）
└──────────────────────────────────────────────────────────┘
```

### 已对齐（相对顶级「可交付」基线）

1. 场景卡稳定路由，不再 UGC/广告/写真串味  
2. 渠道画幅硬化（9:16 / 16:9）  
3. 一条成片 + 字幕 + 片尾 CTA（Creatify/HeyGen 交付形态）  
4. 跨镜 `identity_from` hero 锚定  
5. 口播响度 ≈ -16 LUFS + Edge Neural 分性别音色  

### 仍明显落后

| 差距 | 顶级做法 | 我们现状 | 优先级 |
|------|----------|----------|--------|
| 口型/微表情 | HeyGen Avatar IV | Kling Avatar 可用但穿帮 | P0 |
| 商用 BGM / 混音 | Creatify 曲库 + ducking | 程序化软床 | P0 |
| 字幕样式 / 烧录质量 | CapCut 花字模板 | 底栏白字 | P1 |
| 跨镜人脸/产品锁 | IP-Adapter / 角色库 | 单 hero i2v | P0 |
| UGC 创作者可信度 | 真人库 + 声音克隆 | 生成脸偏假 | P1 |
| 短剧表演与对白 | ShortMax 剧本引擎 | 情绪提示弱 | P1 |
| 批量创意变体 | Advantage+ 工厂 | 单 brief 单路径 | P1 |
| SLA / 失败重试 UX | 旗舰平台 | 依赖 KIE 队列 | P2 |

## 5. 下一轮优化路线（建议严格按序）

### P0 — 继续对标（下一迭代）

1. **上游竖屏生成**：UGC/短剧 hero 与 i2v 必须原生 9:16（本轮 FINAL 靠 pad，分镜仍 1:1）

4. **包装升级**：可配置 BGM 库 + 人声侧链压低；CTA 模板（品牌色/Logo 位）
